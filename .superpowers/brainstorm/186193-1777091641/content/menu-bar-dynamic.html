Dynamic Menu Bar

The menu bar already detects context (Pane / Editor / Terminal). How should the menus visually adapt when context changes? Showing the same bar in 3 contexts.

Swap Menus by Context

Each context gets its own complete menu set. Editor replaces File/Edit/View/Go with File/Edit/Format/View. Terminal gets File/Shell/View. Clean separation — no bleed between contexts.

B

Same Menus, Context Badge + Dimmed Irrelevant

Menus stay consistent (File/Edit/View/Go/Tools), but items irrelevant to the current context are dimmed. The context badge makes it clear what mode you're in. Zero learning curve.

Base Menus + Context Injection

Core menus (File/View/Tools) always present. Context-specific menus are injected between them when active — Editor adds "Edit" and "Format", Terminal adds "Shell". Menus expand and contract naturally.
